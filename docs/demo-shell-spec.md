# MedSpa Review Follow-Up — Demo Shell Spec

## Goal
Create a polished, fake-real MVP shell that feels like a usable product and can be shown without calls or custom walkthrough videos.

## Demo Purpose
- show the workflow clearly
- make the product feel real
- support self-serve exploration
- support a guided product tour

## Demo Account
### Sample business
Luna Aesthetics Med Spa

### Example campaign
Post-Visit Google Review Request

## Demo Screens
1. Dashboard
2. Review Settings
3. Contacts
4. Import Contacts
5. Campaigns
6. Campaign Detail / Message Setup
7. Activity / Results
8. Performance snapshot with Google rating + review count + review growth
9. Template library preview

## Guided Walkthrough Steps
1. Welcome to the product
2. Set your Google review link
3. Import recent clients
4. Create a review request campaign
5. Customize the message
6. Schedule follow-up
7. Track sent requests and follow-ups

## Product Feel
- polished
- premium
- niche-specific
- believable enough to feel like near-real software

## Demo Rules
- use seeded med spa sample data
- clearly functional-feeling UI
- no broken or empty dead ends
- limit feature set to the narrow core workflow
